## Decisions Made

### Decision 1: Use Expo Go for iOS Testing (Not Expo Dev Client)
- **Context**: Need to test iOS without macOS for daily development
- **Options Considered**:
  1. Expo Go - Free, instant QR code testing, limited to Expo SDK APIs
  2. Expo Dev Client - Custom builds, full native module support, requires EAS Build
  3. Full native builds - Maximum control, requires macOS or cloud builds
- **Decision**: Start with Expo Go for MVP development, switch to Dev Client only when native modules required (Week 7+ for Whisper, Week 13+ for audio player)
- **ADR Created**: No (documented in ios-expo-go-setup.md)

### Decision 2: Android Emulator as Primary Development Target
- **Context**: Windows environment, need fast iteration cycles
- **Options Considered**:
  1. Android emulator - Native Windows support, hardware acceleration
  2. iOS Expo Go only - Slower iteration, phone required
  3. Web preview - Fastest but misses mobile-specific features
- **Decision**: Android emulator for 80% of daily work, iOS Expo Go for verification
- **ADR Created**: No (standard practice for Windows React Native dev)

### Decision 3: 1x1 Pixel Placeholder Assets
- **Context**: Expo requires icon/splash assets but we don't have final designs yet
- **Options Considered**:
  1. Generate minimal 1x1 pixel PNGs - Smallest possible, works
  2. Create temporary branded assets - Time consuming, will be replaced
  3. Skip assets, handle errors - Causes Expo warnings/crashes
- **Decision**: 1x1 pixel placeholders with clear README for replacement
- **ADR Created**: No (temporary development decision)
