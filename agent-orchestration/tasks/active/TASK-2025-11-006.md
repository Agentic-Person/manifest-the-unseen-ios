## Objective
Set up ESLint, Prettier, and Husky git hooks to ensure code quality and consistency across the project.

## Context
**Why this task is important**:
- Enforces code quality standards from Day 1
- Catches TypeScript errors before commits
- Maintains consistent code formatting
- Prevents bugs through linting rules

## Implementation Steps

1. **Configure ESLint** (0.5h)
   ```bash
   npm install --save-dev @typescript-eslint/parser @typescript-eslint/eslint-plugin
   npm install --save-dev eslint-plugin-react eslint-plugin-react-native
   ```
   - Create `.eslintrc.js` with React Native + TypeScript rules
   - Add scripts to `package.json`: `"lint": "eslint . --ext .ts,.tsx"`

2. **Configure Prettier** (0.5h)
   ```bash
   npm install --save-dev prettier eslint-config-prettier
   ```
   - Create `.prettierrc`:
     ```json
     {
       "semi": true,
       "singleQuote": true,
       "trailingComma": "es5",
       "printWidth": 100,
       "tabWidth": 2
     }
     ```
   - Add script: `"format": "prettier --write \"**/*.{ts,tsx,js,jsx,json,md}\""`

3. **Set Up Husky Git Hooks** (0.5h)
   ```bash
   npm install --save-dev husky lint-staged
   npx husky install
   npx husky add .husky/pre-commit "npx lint-staged"
   ```
   - Configure `lint-staged` in `package.json`:
     ```json
     "lint-staged": {
       "*.{ts,tsx}": ["eslint --fix", "prettier --write"],
       "*.{json,md}": ["prettier --write"]
     }
     ```

4. **VS Code Settings** (0.5h)
   - Create `.vscode/settings.json`:
     ```json
     {
       "editor.formatOnSave": true,
       "editor.defaultFormatter": "esbenp.prettier-vscode",
       "editor.codeActionsOnSave": {
         "source.fixAll.eslint": true
       }
     }
     ```
   - Create `.vscode/extensions.json` with recommended extensions
