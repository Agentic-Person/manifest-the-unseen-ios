## Objective
Create comprehensive design token system (colors, typography, spacing, shadows) that will be used across the entire Manifest the Unseen app.

## Context
**Why this task is important**:
- Establishes visual consistency across the entire app
- Enables rapid UI development with reusable design primitives
- Ensures brand identity (purple/gold theme) is consistently applied
- Makes future design changes easy (change tokens, propagate everywhere)

**Related documentation**:
- PRD Section 13.1: Visual Design (purple/gold brand, spiritual aesthetic)
- TDD Section 4.2: Design System & Styling (NativeWind + Tailwind)
- CLAUDE.md: Tech Stack - NativeWind (Tailwind CSS for React Native)

**Prerequisite knowledge**:
- NativeWind (Tailwind CSS for React Native)
- React Native StyleSheet
- Accessibility (WCAG AA color contrast)
- Responsive design principles

## Acceptance Criteria
- [x] **Functional**: All design token files created and exportable
- [x] **Colors**: Primary (purple/gold), semantic (success/error/warning), neutral grays
- [x] **Typography**: Font scale (h1-h6, body, caption) with line heights
- [x] **Spacing**: 4px base grid (4, 8, 12, 16, 24, 32, 48, 64, 96)
- [x] **Shadows**: Elevation system (0-5 levels) for cards/modals
- [x] **Accessibility**: All color combinations meet WCAG AA contrast (4.5:1)
- [x] **Documentation**: Usage examples and guidelines documented
- [x] **Code Quality**: TypeScript types, exports organized

## Implementation Steps

1. **Setup Theme Structure** (1h)
   - Create `mobile/src/theme/` directory
   - Set up index.ts for centralized exports
   - Configure TypeScript types for theme

2. **Define Color Palette** (2h)
   - Create `colors.ts`:
     - Primary: Purple shades (50-900)
     - Secondary: Gold shades (50-900)
     - Semantic: success (green), error (red), warning (orange), info (blue)
     - Neutral: Grays (50-900) for text/backgrounds
     - Special: Transparent, white, black
   - Test color contrast with WebAIM tool
   - Document color usage (when to use which)

3. **Define Typography Scale** (2h)
   - Create `typography.ts`:
     - Font families (primary: System, monospace for code)
     - Font sizes (h1: 32px down to caption: 12px)
     - Line heights (1.2 for headings, 1.5 for body)
     - Font weights (400 regular, 600 semibold, 700 bold)
     - Letter spacing
   - Make responsive (scale for different screen sizes)

4. **Define Spacing System** (1h)
   - Create `spacing.ts`:
     - Base unit: 4px
     - Scale: 1 (4px), 2 (8px), 3 (12px), 4 (16px), 6 (24px), 8 (32px), 12 (48px), 16 (64px), 24 (96px)
     - Named spacing: xs, sm, md, lg, xl, 2xl, 3xl
   - Document usage patterns (padding vs margin)

5. **Define Shadow/Elevation System** (1h)
   - Create `shadows.ts`:
     - 6 elevation levels (none, xs, sm, md, lg, xl)
     - iOS-style shadows (shadowColor, shadowOffset, shadowOpacity, shadowRadius)
     - Android-style elevation values
   - Test on both platforms

6. **Create Theme Index** (1h)
   - Centralize all exports in `theme/index.ts`
   - Create theme object for easy consumption
   - Add TypeScript types for autocomplete
   - Document theme structure
